Hooks

Cos'è un Hooks
Gli Hooks ti permettono di utilizzare state ed altre funzioni di React senza dover scrivere una classe. Gli Hooks che andremo a visionare in questa guida saranno:
- UseState
- UseEffect
- UseRef
Cos'è useState
UseState come ho detto prima è un Hooks che ci permette di andar aggiornare il nostro componente. Per usare questo Hooks dovremmo prima andarlo ad importare e poi digitare:
const [Per vedere cosa c'e all'interno dello stato, Serve per modificare lo stato (è UNA FUNZIONE)] = useState("") --> memorizzo solo UN dato!(Più dati = [])
nella quadre si andranno a mettere due parametri divisi da una virgola = useState("") se andiamo a mettere i doppi apici come scritto sopra memorizza un solo dato mentre con le [] andrà a memorizzare più dati.
Esempio di useState:
                        
    import React, { useState } from 'react'

    const App =()=>{
        const [MsgPera, setMsgPera] = useState([])
    
        const URL = "https://raw.githubusercontent.com/icobasco/sample_data_files/master/pera_misure_sample.json"
    
        const scarica =()=>{
            fetch(URL)
            .then((testo)=>testo.json())
            .then((pera)=>{
                setMsgPera(pera[0].data.sensor1.lowRes.humidity + " " + pera[0].data.sensor1.lowRes.temperature)
            
            })
        }
    
        return (
            <div>{MsgPera} Prova{scarica()}</div>
        )
    }
Cos'è useEffect
UseEffect come ho detto prima è un Hooks, sostanzialmente quando usi questo Hooks stai dicendo a react di eseguire la tua funzione "effetto" dopo aver applicato i cambiamenti al DOM. Nell' esempio che andremo a vedere useremo questo useEffect per non far ripetere all'infito i nosti dati, andando nel pratico: useEffect(() => { //cosa chiamare }, [//quando chiamare(1 volta)])
Esempio di useEffect:
                        
    import React, { useState, useEffect } from 'react'

    const App =()=>{
        const [arrayPera, setArrayPera] = useState([])
    
        useEffect(() => {
            scarica()
        }, [])
    
        const URL = "https://raw.githubusercontent.com/icobasco/sample_data_files/master/pera_misure_sample.json"
    
        const scarica =()=>{
            fetch(URL)
            .then((testo)=>testo.json())
            .then((pera)=>{
                pera.map((elemento, indice)=>{
                    setArrayPera((arrayPera)=>[...arrayPera, <div>{elemento.data.sensor1.lowRes.humidity + " " + elemento.data.sensor1.lowRes.temperature + " " }</div>])
                })
            })
        }
    
        return(
            <div>{arrayPera}</div>
        )
    }
Cos'è useRef
UseRef, come ho detto prima è un Hooks, che ci permette di andar a prendere tutti gli attributi cha ha div, ad esempio, lo useRef si usa solitamente al posto di querySelector. Dopo aver importato lo stato, la sintassi per lo useRef è questa: const nomeVar = useRef(null) da come puoi intuire nomeVar è il nome che scegli di dare al tuo Ref e imposti il valore iniziale a null e se in tal caso voui prendere il testo specifico di un div farai: nomeVar.current.textContent.
Esempio di useRef:
                        
    import React, {useRef} from 'react'
    import './App.css'
    
    const App =()=>{
        const prendiTesto = useRef(null)
    
        const ValoreRef =()=>{
            console.log(prendiTesto)
        }
    
        return(
        <div>
            <div ref={prendiTesto}>ciao testo</div>
            <button onClick={ValoreRef}>Prendi Valore Ref</button>
        </div>
        )
    }
    
    export default App

Componenti

Cos'è un componente

Un componente semplicemente ti permette di suddividere l'interfaccia dell'utente del tuo progetto in pezzi piccoli, così da rendere il codice più leggibile e funzionale.

Esempio componenti
                        
 App.js 
 
                        
    import './App.css'
    import Header from "./Header.js"
    
    const App =()=>
    
        return (
        <div>
            <Header/>
        </div>
        )
    }
    
    export default App

Tipi Di Target

Cosa sono i Target?

In React, event.target si riferisce all'elemento DOM che ha scatenato l'evento. Ci sono diversi tipi di event.target in React, ecco una breve introduzione per tutti quelli che andremo a visionare in questa guida:
- event.target
- event.currentTarget

Target

In React, event.target è una proprietà dell'oggetto evento (event) che rappresenta l'elemento del DOM che ha effettivamente scatenato l'evento. In altre parole, event.target restituisce un riferimento all'elemento HTML su cui è stato effettivamente verificato l'evento.

Esempio event.target
                    
    import React from 'react'
    import './App.css'
    
    const App =()=>{
    
        const evento =(event)=>{
            console.log(event.target)
        }
    
        return(
            <div>
                <div onClick={evento}>
                    <div>noi siamo </div>
                    <div>home </div>
                </div>
            </div>
        )
    }
    
    export default App
currentTarget

In React, event.currentTarget è una proprietà dell'oggetto evento (event) che rappresenta l'elemento su cui è stato associato l'evento. A differenza di event.target, che restituisce l'elemento che ha effettivamente scatenato l'evento, event.currentTarget restituisce sempre lo stesso elemento, indipendentemente da dove l'evento è stato scatenato all'interno di esso.

Esempio event.currentTarget
                    
    import React from 'react'
    import './App.css'
    
    const App =()=>{
    
        const evento =(event)=>{
            console.log(event.currentTarget)
        }
    
        return(
            <div>
                <div onClick={evento}>
                    <div>noi siamo </div>
                    <div>home </div>
                </div>
            </div>
        )
    }
    
    export default App
